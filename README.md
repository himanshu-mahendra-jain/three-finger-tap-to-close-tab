# Three Finger Tap to Close Tab

A lightweight browser extension for quickly closing the current tab with
a three-finger tap, middle click, or toolbar click.

The extension is designed to be simple, fast, private, and predictable.
It has no popup, no settings page, no analytics, no tracking, and no
unnecessary browser permissions.

## Supported Browsers

The extension supports:

-   Google Chrome
-   Microsoft Edge
-   Mozilla Firefox

Chrome and Edge use the Chromium manifest. Firefox uses a separate
manifest for browser-specific background execution and add-on metadata.

The JavaScript source code is shared across all supported browsers.

## Features

-   Close the current tab with a three-finger tap
-   Close the current tab with a middle click
-   Close the current tab by clicking the extension toolbar icon
-   Preserve normal middle-click behavior on links
-   Avoid closing tabs while interacting with editable fields
-   Support event detection through Shadow DOM event paths
-   No popup or settings page
-   No storage or browsing-history collection
-   No network requests
-   No analytics or tracking

## Usage

### Three-Finger Tap

Three-finger tap on a webpage outside links and editable fields to close
the current tab.

The touchpad and operating system must be configured so that a
three-finger tap generates a middle click.

### Middle Click

Middle-click on a webpage outside links and editable fields to close the
current tab.

Middle-clicking a link preserves the browser's normal behavior instead
of closing the tab.

### Toolbar

Click the extension icon to close the current tab.

The toolbar action performs the same tab-closing operation as the
three-finger tap and middle-click actions.

## manifest.json

Defines the extension metadata, content script, background service
worker, and toolbar action for Chrome and Edge.

The content script runs on supported webpages so it can detect
middle-click events generated by a physical middle mouse button or a
touchpad three-finger tap configured as a middle click.

## manifest-firefox.json

Defines the Firefox-specific extension configuration while using the
same source files and functionality.

Firefox uses `background.scripts` for background execution.

## src/background.js

Handles tab-close messages from the content script and toolbar icon
clicks.

When the content script requests a tab close, the background script uses
the sender tab ID. When the toolbar icon is clicked, it closes the tab
associated with the toolbar action.

## src/content.js

Listens for auxiliary click events and handles middle-button actions.

The script:

-   Ignores buttons other than the middle mouse button
-   Preserves normal middle-click behavior on links
-   Avoids closing the tab while interacting with editable fields
-   Checks the composed event path for elements inside Shadow DOM
-   Prevents the handled middle-click event from continuing after a
    tab-close action
-   Sends a tab-close request to the background script

## Link Handling

Middle-clicking a link preserves the browser's normal link behavior.

This means a physical middle click or three-finger tap directly on a
link may open the link in a new tab instead of closing the current tab.

The extension cannot reliably distinguish between a physical middle
click and a three-finger tap when both are reported by the browser as
the same middle-button event.

## Editable Fields

The extension avoids closing the current tab when the interaction occurs
within supported editable elements or when an editable element is
active.

The detection includes:

-   Input fields
-   Text areas
-   Select elements
-   Editable content regions
-   Elements with a textbox role

This reduces accidental tab closures while entering or editing content.

## Shadow DOM Handling

The extension checks the event's composed path rather than relying only
on the direct event target.

This improves link and editable-element detection on modern websites
that use Shadow DOM.

## Permissions

The extension does not request optional browser permissions through the
`permissions` field.

The content script uses broad webpage matching because detecting a
three-finger tap or middle click requires the listener to be present on
webpages where the gesture is used.

The extension does not use this access to collect, store, transmit, or
analyze webpage content.

## Privacy

The extension:

-   Does not collect user data
-   Does not collect browsing history
-   Does not use analytics
-   Does not use tracking
-   Does not make network requests
-   Does not store browsing activity
-   Does not store per-site preferences
-   Does not use remote code
-   Does not load external JavaScript

All tab-closing operations happen locally in the browser.

## Performance

The extension is event-driven and performs no continuous work while
idle.

There is no polling, `setInterval`, `MutationObserver`, automatic page
scanning, or background network activity.

The content script registers a single event listener and performs work
only when an auxiliary click event occurs. The background script
responds only to tab-close messages and toolbar clicks.

## Security Design

The extension follows a minimal-capability approach and intentionally
avoids remote code, external dependencies, analytics SDKs, advertising
SDKs, remote configuration, storage access, native messaging, background
network activity, and unnecessary browser permissions.

The content script sends only a fixed tab-close action message. The
background script closes only the sender tab for content-script requests
or the tab associated with a toolbar click.

## Build

The project includes a build script that creates separate distribution
packages for Chrome, Edge, and Firefox.

### Prerequisites

The build process requires:

-   Bash
-   zip

Check whether `zip` is installed:

``` bash
zip -v
```

If it is not installed, install it using your system package manager.

Examples:

``` text
Debian/Ubuntu: sudo apt install zip
```

### Run the Build

Make the build script executable:

``` bash
chmod +x scripts/build.sh
```

Run:

``` bash
./scripts/build.sh
```

The script:

-   Reads the extension name and version from the manifest files
-   Creates a clean `dist/` directory
-   Packages Chrome using `manifest.json`
-   Packages Edge using `manifest.json`
-   Packages Firefox using `manifest-firefox.json` as `manifest.json`
    inside the Firefox ZIP
-   Includes the shared `src/` and `icons/` directories
-   Creates separate ZIP files for each browser

Example output:

``` text
dist/
├── three-finger-tap-to-close-tab-chrome-v1.0.0.zip
├── three-finger-tap-to-close-tab-edge-v1.0.0.zip
└── three-finger-tap-to-close-tab-firefox-v1.0.0.zip
```

The ZIP archives contain only the files required to run the extension.
Repository documentation and store-listing assets are not included in
the distribution packages.

## Known Limitations

-   A three-finger tap must be configured by the operating system or
    touchpad driver to generate a middle click.
-   Browser-protected pages and other restricted pages do not allow
    content scripts to run.
-   A three-finger tap and a physical middle click are handled as the
    same browser input event when both generate a middle-button click.
-   Middle-clicking links preserves the browser's default link behavior
    instead of closing the current tab.
-   Browser interfaces outside webpage content are not handled by the
    content script.
-   Mobile browser support depends on the browser and platform's
    extension capabilities.

## Design Philosophy

Three Finger Tap to Close Tab is intentionally focused on one task:

Close the current tab quickly with a three-finger tap, middle click, or
toolbar click.

The extension favors predictable behavior, minimal permissions, local
processing, and minimal background activity over configuration,
automation, tracking, or unnecessary features.
